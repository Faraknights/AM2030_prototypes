The input is a JSON array of user preference objects, where each object has:
- "user": always "I"
- "reaction": either "like" or "dislike"
- "entities": a dictionary of entity types and their values, e.g., "music", "artist", "podcast", etc.

---

### Rules for Conversion

1. Quantify only entities that exist.
   - If a "music" entity exists, create a music variable x and include Music(x) predicate.
   - If an "artist" entity exists, create an artist variable y and include Artist(y) predicate.
   - Do NOT invent a song if only an artist is mentioned.
2. Link music to artist only if both exist:
   created_by(x, y)
3. User reaction predicates:
   - use the user's reaction as a predicate:
     like(I, v) or dislike(I, v), where v is the variable representing the entity
4. Combine multiple preferences with ∧.
5. Use parentheses to clearly define the scope of all existential quantifiers.
   - Example: ∃x (Song(x) ∧ Dumb(x) ∧ like(I, x))
   - Never write ∃x Song(x) ∧ Dumb(x) ∧ like(I, x)
6. For each entity, use its name as a predicate applied to the variable.
   - Example: If music = "Happy", create a variable x, then include Music(x) ∧ Happy(x)
   - If artist = "Nirvana", create y and include Artist(y) ∧ Nirvana(y)
   - Multi-word entities should replace spaces with underscores for predicates
     e.g., "Burger King" → Burger_King(b)
